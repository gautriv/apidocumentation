Exercise 3: Documenting API Endpoints

Scenario

You are documenting a Library Management API that allows users to manage books, patrons, and borrowing records for a library system.

Your Task

Document the following book-related endpoints of the API:

  • GET /books - List all books
  • GET /books/{id} - Get a specific book
  • POST /books - Add a new book
  • PUT /books/{id} - Update a book
  • DELETE /books/{id} - Delete a book

For each endpoint, include:

  • Description of what the endpoint does
  • Request parameters (path, query, and body as applicable)
  • Sample request
  • Sample response
  • Possible status codes

API Information

The base URL for the API is: https://api.library.com/v1

Sample Book Object:
{
  "id": "123",
  "title": "The Great Gatsby",
  "author": "F. Scott Fitzgerald",
  "isbn": "9780743273565",
  "publication_year": 1925,
  "genre": "Fiction",
  "available": true,
  "copies": 3
}
Bonus Challenge

Document pagination for the GET /books endpoint and explain how users can filter books by genre or availability.

Library Management API: Book Endpoints

Base URL: https://api.library.com/v1

All endpoints require authentication via Bearer token.

3. Add a New Book

Method POST
Endpoint /books
Description Adds a new book to the library collection.
Request Body Parameters
Parameter Type Required Description
title String Yes The title of the book
author String Yes The author of the book
isbn String Yes International Standard Book Number
publication_year Integer Yes Year the book was published
genre String Yes Genre of the book
copies Integer Yes Number of copies available
description String No Description of the book
publisher String No Publisher of the book
Example Request
POST https://api.library.com/v1/books
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
  "title": "1984",

  "genre": "Dystopian Fiction",
  "copies": 5,

4. Update a Book

Method PUT
Endpoint /books/{id}
Description Updates information for an existing book.
Path Parameters
Parameter Type Description
id String The unique identifier of the book

The request body is the same as for the POST request, but all fields are optional. Only the provided fields will be updated.

Example Request
PUT https://api.library.com/v1/books/123
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json

{
  "copies": 5,
  "available": false
}
Example Response (200 OK)
{
  "id": "123",
  "title": "The Great Gatsby",
  "author": "F. Scott Fitzgerald",
  "isbn": "9780743273565",
  "publication_year": 1925,
  "genre": "Fiction",
  "available": false,
  "copies": 5
}
