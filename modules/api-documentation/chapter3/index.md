  <h2 id="introduction">Introduction</h2>
  
  <p class="intro-text">You've done a fantastic job learning how to <strong>structure and write API documentation</strong> that is clear, concise, and developer-friendly. But let's be honest—<strong>the best way to learn is by doing.</strong></p>

</div>

<div class="chapter-section">
  <h2 id="what-to-expect">What to Expect?</h2>
  
  <p>Whether you're new to API documentation or looking to refine your skills, these exercises will help you <strong>bridge the gap between theory and real-world scenarios.</strong></p>
  
  <p><strong>How It Works:</strong><br>
  The chapter contains <strong>5 hands-on exercises</strong> that simulate <strong>real-world API documentation tasks.</strong></p>
  
  <p>You'll:</p>
  <ul>
    <li>Get an API request (or part of it) and write the documentation.</li>
    <li>Think through how to structure information clearly.</li>
    <li>Compare your response with a provided solution.</li>
  </ul>
  
  <p>The goal? <strong>Help you practice like you're already working as an API technical writer.</strong></p>
</div>
